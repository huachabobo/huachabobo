## Lecia AgentOS

`Lecia AgentOS` 是我目前最核心、也最想被认真介绍的项目。

它不是一个“会聊天”的 bot 包装层，而是一套面向自托管团队的 AgentOS control plane。
我想把入口、runtime、governance、docs、operator workflow 与后续 orchestration，收进同一套清晰、长期可维护的系统设计里。
官网负责建立第一理解，文档负责继续展开运行时、治理与控制面的细节。

- 官网与文档中心: [leciabot-site.huach.top](https://leciabot-site.huach.top/)
- 当前进度: `70%`
- 当前形态: `self-hosted AgentOS control plane`
- 默认气质: `local-first` · `self-hosted` · `governable`
- 默认要求: `safe` · `auditable` · `controllable`

### 已经进入真实形态的部分

- 官网与文档中心已经上线，产品入口和叙事骨架是完整的。
- 核心仓库已经具备真实的 `web` / `server` / `CLI` / `persistence` / `runtime` 切片。
- `approval`、`trust`、`snapshot`、`restore`、`observability` 这些治理能力已经不是概念，而是工程中的默认能力。
- 前台对话、后台任务、provider 路由与执行证据，正在被收进同一条可追踪的运行时链路。

### 还在持续打磨的部分

- 把 controlled core 做得更稳，让它更适合真实部署，而不只是漂亮 demo。
- 继续打磨 provider / adapter runtime，让它能长期运行，而不是短期可演示。
- 强化 operator 视角下的 approval、auditability 与 recovery 体验。
- 把 orchestration layer 继续做实，避免产品退化成单纯的 reply shell。

> `Lecia AgentOS` 核心仓库目前还在持续私有开发中，所以我把官网与文档中心作为当前最主要的对外入口。

<details>
<summary>How I think about systems</summary>

我喜欢把看起来柔软的体验，建立在足够坚实的系统之上。

我会反复追问这些问题：

- 产品有没有清晰的控制面
- 运行时是不是诚实、稳定、可解释
- 文档是不是产品的一部分，而不是最后补上的说明
- 默认能力里有没有治理、审计、恢复与安全边界
- 项目能不能被部署、维护、继续演化
